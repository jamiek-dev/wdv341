PHP Functions

A function is a pre-written piece of code that can be processed repeatedly. This allows for reusability and modularity of code. Information may be passed into a function. This type of information is called a parameter. Functions can accept multiple parameters. The number of parameters and how they will be used are determined when the function is defined or 'written'.

Defining a function

  • PHP keyword function
  • A function can be used/called multiple times.
  • A well written function should only do one primary task. It should contain the code necessary to do that one task.
  • The name of the function should be very descriptive in order to be self documenting.
  • In a Class or an Object a function is often referred to as a “Method”. All functions are written the same way in a Class as in PHP.
  • In general do not overload functions in PHP. Instead use ‘optional’ parameters as needed to provide flexibility.
  • Function names are NOT case sensitive

Example of a function definition:

function function_name() {
    // Code to be executed when this function is called
}

Calling a function

  • PHP may call a function by using the name of the function as a command 
    calculate_sales_total();
  • A PHP object may call a function "method" that is part of the object 
    $month_to_date_sales->calculate_sales_total();
  • PHP may call a static function from the class without using an object by accessing the Class. 
    Class_Name::function_name();

Parameters / arguments of a function

  • A function may be written to accept one or more parameters or arguments when it is defined
  • Place a variable name within the ( ) of the function definition for each parameter the function is allowed to process.
  • Separate parameters/variable names with a ',' if there are more than one for the function.
  • When calling a function with one or more parameters pass the required number of variables within the ( ) of the function 
    $process_month = 'July';
display_month($process_month); // Calls the function and passes 'July' into it to be used

    Example of a function definition with one parameter/argument

    function display_month($month) {
    echo $month;
}
  • When calling a function with one or more parameters pass the required number of variables within the ( ) of the function 
    $starting_month = 'July';
$last_process_month = 'October';
display_time_period($starting_month, $last_process_month);

    Example of a function definition with two parameters/arguments

    function display_time_period($start_month, $end_month) {
    echo "$start_month through $end_month"; // Will output as a rendered string
}

Returning Values from a function

  • Functions can 'return' a processed value back to where it was called.
  • This is useful for getting information from an object or having a process performed on a piece of data as part of several steps.
  • Use the keyword return and a variable name when defining the function.
  • A function can only return one value.
    • Note: It may however return an array or an object which may contain multiple values.
  • The returned value must be captured in a variable or processed at the point it was called.

    Example of a function definition that returns a value instead of displaying the result.

    function get_page_count() {
    return $page_count; // Returns the current value of this variable
}

    Example of the call to the function. The value of the returned variable will be placed 

    <h4>Page Number: <?=get_page_count()?></h4>

Variable Scope

Variable Scope is used to determine whether or not a variable is available to be used. The Scope of a variable is determined by how and where it is defined. PHP has two levels of variable scope.

  • Local Scope
    • A variable defined within a function has local scope.
    • It is only available to the function.
    • It cannot be used outside of the function.
    • It will be destroyed as soon as the function is complete.
      • Note: The PHP keyword static can be used to define a local variable that retains its value after the function is complete
  • Global Scope
    • A variable defined within the page but NOT within a function has global scope.
    • This variable is available anywhere within the page of the application.
    • Generally global scope variables are defined near the top of the application page.
    • A global variable can be used anwhere on the page, including by and within functions called on that page.
    • The PHP keyword global is used within function definitions to tell the function to access a global scope variable

Terminlogy:

  • Function This is a set of code that can be processed/called multiple times by the same page.
  • Method Another name for function. Often associated with functions defined as part of a Class or object.
  • Scope This determines where a function is available to be used within an application. Depends upon where the variable is defined.
  • Local A variable defined within a function has local scope and can only be used within that function.
  • Global A variable defined outside of a function has global scope and can be used anywhere within the page including other functions.
  • Static A keyword that allows a local variable to retain its value once the function has completed.
